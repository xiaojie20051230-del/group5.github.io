目录

什么是模板继承

在开发博客时，每个页面都有重复的结构：HTML 文档声明、<head> 中的 SEO 标签、顶部导航栏、底部页脚。如果每个页面都复制粘贴这些内容，维护将是一场噩梦。模板继承的作用就是：定义一个基础骨架，子页面只填充变化的部分。

Jekyll 的继承链

我们的博客采用了三级继承结构：

default.html（基础骨架）
    │
    ├── post.html（文章布局）
    │       └── 被 23 篇文章使用
    │       └── 包含：标题、目录、元信息、正文、标签、上下篇导航
    │
    └── page.html（页面布局）
            └── 被 about / categories 使用
            └── 包含：简单标题 + 内容

父模板：default.html

default.html 负责定义所有页面的公共结构：

<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>{% if page.title %}{{ page.title }} - {{ site.title }}{% else %}{{ site.title }}{% endif %}</title>
  {% seo %}
  <link rel="stylesheet" href="{{ '/assets/css/custom.css' | relative_url }}">
</head>
<body>
  <header class="site-header">...</header>
  <main class="page-content">
    {{ content }}
  </main>
  <footer class="site-footer">...</footer>
</body>
</html>

关键语法：{{ content }} 是一个占位符，子模板的内容会被注入到这个位置。

子模板：post.html

文章页面需要特殊的布局：左侧目录、标题元信息、标签、上下篇导航。它通过 Front Matter 声明继承关系：

---
layout: default
---

然后在 {{ content }} 的位置填充自己的结构：

<article class="post">
  <header class="post-header">
    <h1 class="post-title">{{ page.title }}</h1>
    <div class="post-meta">
      <span>{{ page.date | date: "%Y年%m月%d日" }}</span>
      <span>{{ page.author }}</span>
    </div>
  </header>

  <div class="post-body">
    <aside class="toc-sidebar">...</aside>
    <div class="post-content">
      {{ content }}
    </div>
  </div>
</article>

注意这里的双重 content：

  • 第一个 content 是 post.html 注入到 default.html 的整体内容
  • 第二个 content 是 Markdown 文章的正文注入到 post.html 的位置

当 Jekyll 处理一篇 Markdown 文章时，完整的渲染流程如下：

┌─────────────────┐
│  Markdown 源文件 │  ← 包含 Front Matter 和正文
└────────┬────────┘
         │ 1. 解析 Front Matter，确定使用 post.html
         ▼

      <h2 id="问题背景">问题背景</h2>

<p>静态博客没有数据库，也没有后端服务器。当文章数量增加到 20+ 篇时，读者如何快速找到想要的内容？传统的方案是依赖分类和标签，但这需要读者对内容结构有一定了解。<strong>搜索功能</strong>成为提升用户体验的关键。</p>

<h2 id="核心思路预生成索引--客户端查询">核心思路：预生成索引 + 客户端查询</h2>

<p>静态博客实现搜索的核心公式可以概括为：</p>

\[\text{搜索} = \text{预生成索引} + \text{前端加载} + \text{客户端模糊匹配}\]

<p>这与传统动态网站的搜索架构形成鲜明对比：</p>

<table>
  <thead>
    <tr>
      <th>维度</th>
      <th>动态网站搜索</th>
      <th>静态博客搜索</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>后端服务</td>
      <td>需要</td>
      <td>不需要</td>
    </tr>
    <tr>
      <td>数据库</td>
      <td>MySQL/Elasticsearch</td>
      <td>无</td>
    </tr>
    <tr>
      <td>实时性</td>
      <td>实时查询</td>
      <td>构建时生成索引</td>
    </tr>
    <tr>
      <td>维护成本</td>
      <td>高</td>
      <td>极低</td>
    </tr>
    <tr>
      <td>适用规模</td>
      <td>大型站点</td>
      <td>中小型博客</td>
    </tr>
  </tbody>
</table>

<p>Jekyll 构建时，我们用 Liquid 模板遍历所有文章，生成一个 JSON 文件：</p>

<p>关键点：</p>
<ul>
  <li><code class="language-plaintext highlighter-rouge">strip_html</code> 去除 Markdown 中的 HTML 标签，只保留纯文本</li>
  <li><code class="language-plaintext highlighter-rouge">truncate: 300</code> 限制摘要长度，控制索引文件体积</li>
  <li>索引文件大小直接影响首次加载速度</li>
</ul>

<h2 id="第二步前端加载索引">第二步：前端加载索引</h2>

<p>页面加载后，通过 <code class="language-plaintext highlighter-rouge">fetch()</code> 异步获取索引：</p>

<h2 id="第三步fusejs-模糊匹配">第三步：Fuse.js 模糊匹配</h2>

<p><a href="https://fusejs.io/">Fuse.js</a> 是一个轻量级的模糊搜索库，支持以下特性：</p>

<ul>
  <li><strong>容错搜索</strong>：输入 “jekll” 也能匹配 “jekyll”</li>
  <li><strong>多字段权重</strong>：标题匹配的优先级高于正文内容</li>
  <li><code class="language-plaintext highlighter-rouge">threshold: 0.3</code>：值越小，匹配越严格；值越大，结果越宽松</li>
</ul>

<p>搜索执行代码：</p>

<div class="language-javascript highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="kd">function</span> <span class="nf">performSearch</span><span class="p">(</span><span class="nx">query</span><span class="p">)</span> <span class="p">{</span>
  <span class="k">if </span><span class="p">(</span><span class="o">!</span><span class="nx">query</span> <span class="o">||</span> <span class="nx">query</span><span class="p">.</span><span class="nx">length</span> <span class="o">&lt;</span> <span class="mi">2</span><span class="p">)</span> <span class="k">return</span> <span class="p">[];</span>
  <span class="k">return</span> <span class="nx">fuse</span><span class="p">.</span><span class="nf">search</span><span class="p">(</span><span class="nx">query</span><span class="p">).</span><span class="nf">map</span><span class="p">(</span><span class="nx">r</span> <span class="o">=&gt;</span> <span class="nx">r</span><span class="p">.</span><span class="nx">item</span><span class="p">);</span>
<span class="p">}</span>
</code></pre></div></div>

<h3 id="防抖处理">防抖处理</h3>

<p>用户每输入一个字符就触发搜索会造成性能浪费。我们加入 200ms 防抖：</p>

<div class="language-javascript highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="kd">let</span> <span class="nx">debounceTimer</span><span class="p">;</span>
<span class="nx">searchInput</span><span class="p">.</span><span class="nf">addEventListener</span><span class="p">(</span><span class="dl">'</span><span class="s1">input</span><span class="dl">'</span><span class="p">,</span> <span class="p">(</span><span class="nx">e</span><span class="p">)</span> <span class="o">=&gt;</span> <span class="p">{</span>
  <span class="nf">clearTimeout</span><span class="p">(</span><span class="nx">debounceTimer</span><span class="p">);</span>
  <span class="nx">debounceTimer</span> <span class="o">=</span> <span class="nf">setTimeout</span><span class="p">(()</span> <span class="o">=&gt;</span> <span class="p">{</span>
    <span class="kd">const</span> <span class="nx">results</span> <span class="o">=</span> <span class="nf">performSearch</span><span class="p">(</span><span class="nx">e</span><span class="p">.</span><span class="nx">target</span><span class="p">.</span><span class="nx">value</span><span class="p">);</span>
    <span class="nf">renderResults</span><span class="p">(</span><span class="nx">results</span><span class="p">);</span>
  <span class="p">},</span> <span class="mi">200</span><span class="p">);</span>
<span class="p">});</span>
</code></pre></div></div>

<h2 id="总结">总结</h2>

<p>静态博客的搜索方案证明了：<strong>没有后端，也能实现不错的搜索体验</strong>。这种架构特别适合个人博客、文档站点等中小型内容场景。如果未来文章数量增长到数百篇，可以考虑：</p>

<ol>
  <li>分页加载索引（按分类拆分多个 JSON 文件）</li>
  <li>接入 Algolia DocSearch 等专业搜索服务</li>
  <li>使用 Web Worker 在后台线程执行搜索，避免阻塞 UI</li>
</ol>

      <h2 id="问题背景">问题背景</h2>

<p>静态博客没有数据库，也没有后端服务器。当文章数量增加到 20+ 篇时，读者如何快速找到想要的内容？传统的方案是依赖分类和标签，但这需要读者对内容结构有一定了解。<strong>搜索功能</strong>成为提升用户体验的关键。</p>

<h2 id="核心思路预生成索引--客户端查询">核心思路：预生成索引 + 客户端查询</h2>

<p>静态博客实现搜索的核心公式可以概括为：</p>

\[\text{搜索} = \text{预生成索引} + \text{前端加载} + \text{客户端模糊匹配}\]

<p>这与传统动态网站的搜索架构形成鲜明对比：</p>

<table>
  <thead>
    <tr>
      <th>维度</th>
      <th>动态网站搜索</th>
      <th>静态博客搜索</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>后端服务</td>
      <td>需要</td>
      <td>不需要</td>
    </tr>
    <tr>
      <td>数据库</td>
      <td>MySQL/Elasticsearch</td>
      <td>无</td>
    </tr>
    <tr>
      <td>实时性</td>
      <td>实时查询</td>
      <td>构建时生成索引</td>
    </tr>
    <tr>
      <td>维护成本</td>
      <td>高</td>
      <td>极低</td>
    </tr>
    <tr>
      <td>适用规模</td>
      <td>大型站点</td>
      <td>中小型博客</td>
    </tr>
  </tbody>
</table>

<p>Jekyll 构建时，我们用 Liquid 模板遍历所有文章，生成一个 JSON 文件：</p>

<p>关键点：</p>
<ul>
  <li><code class="language-plaintext highlighter-rouge">strip_html</code> 去除 Markdown 中的 HTML 标签，只保留纯文本</li>
  <li><code class="language-plaintext highlighter-rouge">truncate: 300</code> 限制摘要长度，控制索引文件体积</li>

<h2 id="第三步fusejs-模糊匹配">第三步：Fuse.js 模糊匹配</h2>

<p><a href="https://fusejs.io/">Fuse.js</a> 是一个轻量级的模糊搜索库，支持以下特性：</p>

<ul>
  <li><strong>容错搜索</strong>：输入 “jekll” 也能匹配 “jekyll”</li>
  <li><strong>多字段权重</strong>：标题匹配的优先级高于正文内容</li>
  <li><code class="language-plaintext highlighter-rouge">threshold: 0.3</code>：值越小，匹配越严格；值越大，结果越宽松</li>
</ul>

<p>搜索执行代码：</p>

<div class="language-javascript highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="kd">function</span> <span class="nf">performSearch</span><span class="p">(</span><span class="nx">query</span><span class="p">)</span> <span class="p">{</span>
  <span class="k">if </span><span class="p">(</span><span class="o">!</span><span class="nx">query</span> <span class="o">||</span> <span class="nx">query</span><span class="p">.</span><span class="nx">length</span> <span class="o">&lt;</span> <span class="mi">2</span><span class="p">)</span> <span class="k">return</span> <span class="p">[];</span>
  <span class="k">return</span> <span class="nx">fuse</span><span class="p">.</span><span class="nf">search</span><span class="p">(</span><span class="nx">query</span><span class="p">).</span><span class="nf">map</span><span class="p">(</span><span class="nx">r</span> <span class="o">=&gt;</span> <span class="nx">r</span><span class="p">.</span><span class="nx">item</span><span class="p">);</span>
<span class="p">}</span>
</code></pre></div></div>

  <span class="p">},</span> <span class="mi">200</span><span class="p">);</span>

<h2 id="总结">总结</h2>

<p>静态博客的搜索方案证明了：<strong>没有后端，也能实现不错的搜索体验</strong>。这种架构特别适合个人博客、文档站点等中小型内容场景。如果未来文章数量增长到数百篇，可以考虑：</p>

<ol>

实用技巧

条件判断

在模板中经常需要根据页面类型显示不同内容：

{% if page.layout == 'post' %}
  <div class="post-nav">...</div>
{% endif %}

循环遍历

分类页面需要遍历所有文章并按分类分组：

{% for category in site.categories %}
  <h2>{{ category[0] }}</h2>
  {% for post in category[1] %}
    <a href="{{ post.url }}">{{ post.title }}</a>
  {% endfor %}
{% endfor %}

过滤器

Liquid 提供丰富的过滤器处理数据：

过滤器 作用 示例
date 格式化日期 {{ "2026-05-01" | date: "%Y年%m月%d日" }}
relative_url 生成相对路径 {{ "/about/" | relative_url }}
strip_html 去除 HTML 标签 {{ "<p>text</p>" | strip_html }}
truncate 截断字符串 {{ "long text" | truncate: 10 }}

总结

模板继承让博客的代码高度 DRY（Don’t Repeat Yourself）。我们的博客虽然只有 3 个布局文件，却支撑了 20+ 个页面的渲染。理解这个机制后，新增页面类型（比如标签页、归档页）也变得非常容易——只需新建一个继承 default.html 的布局即可。
